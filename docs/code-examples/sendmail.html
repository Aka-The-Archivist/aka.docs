Email System

[aka.emails] is a powerful npm package that integrates the functionalities of popular mailing libraries such as Node Mailer and SendGrid, enhancing them with a Handlebars (.hbs) templating system. This module is part of the Aka Backend framework, but can also be used independently to provide a robust and flexible email sending solution.

By integrating with Node Mailer and SendGrid, aka.emails brings the best of both worlds into a single, easy-to-use interface. Whether you need to send a few emails or thousands, whether it's a simple text message or a complex HTML layout, aka.emails has you covered.

The integration with Handlebars adds an additional layer of power and flexibility. Handlebars is a popular templating engine that allows you to create dynamic content easily. With aka.emails, you can define your email templates in .hbs files, which can then be filled with data at runtime. This makes it extremely easy to create personalized emails for each recipient.

Here are some key features of aka.emails:​

info

  • Integration with Node Mailer and SendGrid: Leverage the power and flexibility of these popular mailing libraries directly from your Aka Backend application.

  • Handlebars templating: Create dynamic, personalized emails with ease using .hbs templates.

  • Flexibility: aka.emails can be used as a standalone package or as part of the Aka Backend framework, depending on your needs.

  • Ease of use: Despite its power, aka.emails maintains a simple and intuitive interface that makes it easy to send emails, even for developers who are new to Node.js or backend development.

In the following sections of this documentation, we will explore in detail how to install and use aka.emails, including examples of sending both text and HTML emails, creating and using .hbs templates, and more. Whether you're an experienced developer or just getting started, we're confident that aka.emails can make your email-sending tasks easier and more efficient.

Setting Up the Project​

Run the following command in your shell to install the driver in your project directory:

npm install aka.emails@latest

This command performs the following actions:

  • Downloads the aka.emails package and the dependencies it requires
  • Saves the package in the node_modules directory

Records the dependency information in the package.json file

Nodemailer Driver​

Now that our template is ready, we can use it to send an email. Here's how you can do this with the aka.emails module:

/routes/email/node.js
import {NodeMailer} from "aka.emails";

/**
 *
 * @param req
 * @param res
 */
export default async function post(req, res) {

        // create mail instance
        const mail = new NodeMailer('test@example.com');

        // set sendgrid from address
        mail.from('no-reply@example.com')

        // set subject mail
        mail.subject(`🔐 Confirm code to continue authentication`);

        // send async or await email with template
        await mail.send('example.code', {code:3344585}).then((message) =>
            console.log(message)
        );

}

SendGrid Driver​

Now that our template is ready, we can use it to send an email. Here's how you can do this with the aka.emails module:

/routes/email/sendgrid.js
import {SendGridMail} from "aka.emails";

/**
 *
 * @param req
 * @param res
 */
export default async function post(req, res) {

        // create mail instance
        const mail = new SendGridMail('test@example.com');

        // set sendgrid api key
        mail.setApiKey('SENDGRID_API_KEY')

        // set sendgrid from address
        mail.from('no-reply@example.com')

        // set subject mail
        mail.subject(`🔐 Confirm code to continue authentication`);

        // send async or await email with template
        await mail.send('example.code', {code:3344585}).then((message) =>
            console.log(message)
        );

}
